# i18n Design: Arabic + English Bilingual Support

**Date:** 2026-04-24
**Status:** Approved

## Goal

Add Arabic/English language switching to the entire INFINDER frontend. Language preference is stored in `localStorage`. Arabic activates RTL layout. English is the default.

## Approach

Option C: `react-i18next` with a single typed TypeScript translation file per language. All strings live in `locales/en.ts` and `locales/ar.ts`. Components call `t('key')`.

## New Files

### `frontend/src/i18n.ts`
Initializes i18next with:
- `react-i18next` + `i18next-browser-languagedetector` for localStorage detection
- Resources: `en` and `ar` namespaces loaded from `locales/en.ts` and `locales/ar.ts`
- `localStorage` key: `infinder_lang`
- Default language: `en`
- Fallback language: `en`
- On language change: sets `document.documentElement.dir` (`ltr`/`rtl`) and `document.documentElement.lang`

### `frontend/src/locales/en.ts`
TypeScript object exported as `const`. All English UI strings keyed by semantic name (e.g. `nav_dashboard`, `login_title`, `wizard_goal_title`).

### `frontend/src/locales/ar.ts`
Same keys as `en.ts`. All values are accurate Arabic translations. Egyptian dialect where natural; Modern Standard Arabic for formal/financial terms.

## Language Switcher UI

Located in `Navbar.tsx`, right side, before the Sign out button.

```
[ EN 🇬🇧 ]  [ عر 🇪🇬 ]
```

- Active language: `bg-infinder-lime text-infinder-black font-semibold`
- Inactive language: `border border-gray-300 text-gray-600`
- Both are `rounded-full px-2 py-1 text-xs` buttons
- Clicking calls `i18n.changeLanguage(lang)` which also sets `dir` and `lang` on `<html>`

## RTL Handling

- `document.documentElement.dir = 'rtl'` when Arabic is active
- Cairo font loaded via Google Fonts `<link>` in `index.html`
- `[dir=rtl] { font-family: 'Cairo', sans-serif; }` in `index.css`
- Tailwind flex/grid reverse automatically under `dir=rtl`
- Any explicit `left`/`right` spacing utilities (`ml-`, `mr-`, `pl-`, `pr-`) audited and replaced with logical equivalents (`ms-`, `me-`, `ps-`, `pe-`) where they would break RTL

## Translation Key Convention

Keys use `snake_case` with a page/section prefix:

- `nav_` — navbar and bottom nav
- `landing_` — landing page
- `auth_` — login + register shared
- `dashboard_` — dashboard page
- `invest_` — investment options
- `wizard_` — smart assistant wizard
- `chat_` — smart assistant chat mode
- `fund_` — funding page
- `profile_` — profile page
- `reports_` — reports page
- `learn_` — learning modules
- `rewards_` — rewards dashboard
- `admin_` — admin panel
- `kyc_` — KYC review page
- `common_` — shared strings (e.g. "Save", "Cancel", "Back", "Loading")

## Out of Scope

- Backend translations (API error messages remain in English)
- Dynamic content from the database (portfolio names, etc.)
- Per-user language saved to Supabase
